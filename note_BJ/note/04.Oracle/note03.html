列的别名
当一个查询语句中的字段不是表中纯粹的字段
而是一个函数或表达式时，那么查询出来的结果
集中对应的该字段名就是这个函数或表达式，
可读性差，为此可以为这样的字段添加别名。
若希望别名中出现空格或区分大小写,可以用双引
号将其括起来.
SELECT ename,sal*12 "s al"
FROM emp

AND,OR用来连接多个条件
AND优先级高于OR
查看工资高于1000的职位是CLERK和
SALESMAN的员工
SELECT ename, sal, job 
FROM emp  SALESMAN
WHERE sal > 1000 
AND (job = 'SALESMAN'
OR job='CLERK')

LIKE用于模糊匹配字符串
支持两个通配符:
%:任意个字符(0-多个)
_:单一的一个字符

查看姓名第二个字母是"A"的员工
SELECT ename, job 
FROM emp 
WHERE ename LIKE '_A%';


IN(list)与NOT IN(list)
判断在列表中或不在列表中
IN与NOT IN也常用在子查询在中

查看职位是CLERK或SALESMAN的员工信息
SELECT ename,job
FROM emp
WHERE job IN ('CLERK','SALESMAN')


ANY(list)与ALL(list)
联合>,>=,<,<=多个值时使用
>ANY:大于列表之一即可(大于最小的)
>ALL:大于列表所有(大于最大的)
<ANY:小于最大的
<ALL:小于最小的
通常被用在子查询中。

DISTINCT关键字
用于去除重复行

查看公司有职位？
SELECT DISTINCT job
FROM emp

DISTINCT可以对多列去重,去重的原则
是多个字段值得组合没有重复行。
SELECT DISTINCT job,deptno
FROM emp

ORDER BY子句
用于对结果集按照给定字段进行排序
排序分为:
升序:ASC,不写默认就是升序
降序:DESC
ORDER BY只能定义在查询的最后一个子句上

查看公司中工资的排名
SELECT ename,sal,deptno
FROM emp
ORDER BY sal DESC

可以按照多字段排序，每个字段都可以单独
指定排序方式，排序按照从左到右的优先级
顺序排序，即:先按照第一个字段排序结果集
当第一个字段有重复值再按照第二个字段排序
依此类推。
SELECT ename,deptno,sal
FROM emp
ORDER BY deptno DESC,sal DESC

NULL值在排序中被认作最大值
SELECT ename,comm
FROM emp
ORDER BY comm DESC


聚合函数
聚合函数是对结果集进行统计使用的
对字段值的统计有:
MAX,MIN,AVG,SUM
分别求最大值，最小值，平均值和总和

COUNT是对记录数的统计

查看公司工资情况?
SELECT MAX(sal),MIN(sal),
       AVG(sal),SUM(sal)
FROM emp

查看公司一共多少人?
SELECT COUNT(sal)
FROM emp

SELECT NVL(comm,0) FROM emp

聚合函数忽略NULL值
SELECT AVG(NVL(comm,0))
FROM emp

求表中记录总数
SELECT COUNT(*)
FROM emp

GROUP BY子句
GROUP BY子句可以对结果集按照给定
字段值相同的记录进行分组
GROUP BY是配合聚合函数对结果集进行
分组统计使用的。

查看每个部门的平均工资?
SELECT AVG(sal),deptno
FROM emp
GROUP BY deptno

多字段分组
分组的几个字段值都相同的记录被看作一组
查看每个部门每种职位的最高工资?
SELECT MAX(sal),deptno,job
FROM emp
GROUP BY deptno,job

查看部门的平均工资，前提是该部门
平均工资高于2500?
SELECT AVG(sal),deptno
FROM emp
GROUP BY deptno
HAVING AVG(sal)>2500

WHERE与HAVING
WHERE是用来过滤记录的
HAVING是用来过滤分组的
它们过滤的时机也不相同，先WHERE后HAVING
WHERE是在查询表时逐行过滤以选取满足条件的
记录。
HAVING是在数据查询后并且分完组后对分组进行
过滤的，以决定分组的取舍。
HAVING必须跟在GROUP BY后面。

查看平均工资高于2000的那些部门的最高与
最低工资?
SELECT MAX(sal),MIN(sal),deptno
FROM emp
GROUP BY deptno
HAVING AVG(sal)>2000

关联查询
从多张表中关联查询数据，关键点在于
连接的条件，根据该条件数据库才知道
这些表中的数据的对应关系，从而分别
获取这些对应记录中查询的字段并构成
结果集。

查看每个员工的名字以及其所在部门的名字?
SELECT e.ename,e.deptno,d.dname
FROM emp e,dept d
WHERE e.deptno=d.deptno

关联要有连接条件，不添加连接条件会产生
笛卡儿积，这通常是一个无意义的结果集
原则上N张表关联查询至少要有N-1个连接条件

查看SALESMAN所在那个城市？
SELECT e.ename,e.job,d.loc
FROM emp e,dept d
WHERE e.deptno=d.deptno
AND e.job='SALESMAN'

SELECT e.ename,e.job,d.loc
FROM emp e,dept d
WHERE e.deptno=d.deptno
AND d.loc='NEW YORK'

查看每个城市员工的平均工资?
SELECT AVG(e.sal),d.loc
FROM emp e,dept d
WHERE e.deptno=d.deptno
GROUP BY d.loc


内连接
内连接也是关联查询的一种。

查看每个员工以及其所在部门名?
SELECT e.ename,d.dname
FROM emp e JOIN dept d
ON e.deptno=d.deptno
WHERE d.dname='SALES'

关联查询中不满足连接条件的数据不会
被查询出来。

外连接
外连接即可以满足连接条件的记录查询出
来之外，还可以将不满足连接条件的记录
也查询出来。
外连接分为:
左外连接:以JOIN左侧表作为驱动表，该表中
所有记录都要显示，那么来自右侧表的字段值
若不满足连接条件则为NULL。
右外连接:
全外连接:

列出所有员工以及其所在部门名
SELECT e.ename,d.dname
FROM emp e 
 LEFT|RIGHT|FULL OUTER JOIN 
     dept d
ON e.deptno=d.deptno


SELECT e.ename,d.dname
FROM emp e,dept d
WHERE e.deptno=d.deptno(+)



自连接
自连接是当前表的一条数据对应当前表
的多条数据
自连接用于解决相同属性的一组数据，但是
又存在上下级关系的树状结构组织时使用。

查看每个员工以及其上司的名字?

SELECT e.ename,m.ename
FROM emp e,emp m
WHERE e.mgr=m.empno

SELECT e.ename,m.ename
FROM emp e JOIN emp m
ON e.mgr=m.empno

查看FORD的上司在哪个城市?
分别列出:
FORD，上司的名字，以及上司所在城市
SELECT e.ename,m.ename,d.loc
FROM emp e,emp m,dept d
WHERE e.mgr=m.empno
AND m.deptno=d.deptno
AND e.ename='FORD'

SELECT e.ename,m.ename,d.loc
FROM emp e JOIN emp m
ON e.mgr=m.empno
JOIN dept d
ON m.deptno=d.deptno
WHERE e.ename='FORD'
